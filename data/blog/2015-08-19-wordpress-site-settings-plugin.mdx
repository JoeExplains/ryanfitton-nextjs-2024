Download from Github [here](https://github.com/ryanfitton/wp-settings-panel-plugin).

This plugin has been created to make it easy to integrate a 'Website Settings' page into WordPress, this settings page can include different options such as a Telephone Number, Contact Email Address, META Keywords and Footer Information.

Let's start, first download the code from this [GIT Repo hosted on Github](https://github.com/ryanfitton/wp-settings-panel-plugin). Place your downloaded contents into a new folder named 'wp-settings-panel-plugin'. This folder should be within your WordPress plugin folder: '/wp-content/plugins/' folder.

This plugin uses only two files, one for the plugin core and the other for minimal styling.

The 'options.php' file is the core code of the plugin.

Firstly, the plugin META information is shown, along with some comments of how to display the final values.

You will see code to add additional settings and links. You can use these additional links for a Support page, this isn&#8217;t really necessary but may help a user if they are having issues.

Remember you will need to replace the support URL with your own on line 52: &#8216;https://ryanfitton.co.uk/blog/wordpress-site-settings-plugin/&#8217;.

<a href="/static/img/blog/2015/08/ws-plugin-2.png"><img loading="lazy"  src="/static/img/blog/2015/08/ws-plugin-2-1024x613.png" alt="ws-plugin-2" width="1024" height="613" srcset="/static/img/blog/2015/08/ws-plugin-2-1024x613.png 1024w, /static/img/blog/2015/08/ws-plugin-2-300x180.png 300w, /static/img/blog/2015/08/ws-plugin-2.png 1034w" sizes="(max-width: 1024px) 100vw, 1024px" /></a>

Now the &#8216;style.css&#8217; document is included by using the &#8216;wp_enqueue_scripts&#8217; command.

<a href="/static/img/blog/2015/08/ws-plugin-3.png"><img loading="lazy"  src="/static/img/blog/2015/08/ws-plugin-3.png" alt="ws-plugin-3" width="963" height="161" srcset="/static/img/blog/2015/08/ws-plugin-3.png 963w, /static/img/blog/2015/08/ws-plugin-3-300x50.png 300w" sizes="(max-width: 963px) 100vw, 963px" /></a>

Next, yiu will see configuration options for the Admin sidebar. You can find help on the WordPress Codex area for this function <a href="https://codex.wordpress.org/Function_Reference/add_menu_page">here</a>.

<a href="/static/img/blog/2015/08/ws-plugin-4.png"><img loading="lazy"  src="/static/img/blog/2015/08/ws-plugin-4-1024x258.png" alt="ws-plugin-4" width="1024" height="258" srcset="/static/img/blog/2015/08/ws-plugin-4-1024x258.png 1024w, /static/img/blog/2015/08/ws-plugin-4-300x76.png 300w, /static/img/blog/2015/08/ws-plugin-4.png 1328w" sizes="(max-width: 1024px) 100vw, 1024px" /></a>

Now, let&#8217;s setup the settings page content. You can think of this section as a wrapper for the input fields. This code uses callbacks to retrieve the content for this page, these functions are part of the <a href="https://codex.wordpress.org/Settings_API">WordPress Settings API</a>, introduced in version 2.7.

<a href="/static/img/blog/2015/08/ws-plugin-5.png"><img loading="lazy"  src="/static/img/blog/2015/08/ws-plugin-5-1024x462.png" alt="ws-plugin-5" width="1024" height="462" srcset="/static/img/blog/2015/08/ws-plugin-5-1024x462.png 1024w, /static/img/blog/2015/08/ws-plugin-5-300x135.png 300w, /static/img/blog/2015/08/ws-plugin-5.png 1225w" sizes="(max-width: 1024px) 100vw, 1024px" /></a>

Next, we configure and register the setting input field names. We also set the text which is shown on the page with the &#8216;display_header_options_content&#8217; function.

<a href="/static/img/blog/2015/08/ws-plugin-6.png"><img loading="lazy"  src="/static/img/blog/2015/08/ws-plugin-6-1024x373.png" alt="ws-plugin-6" width="1024" height="373" srcset="/static/img/blog/2015/08/ws-plugin-6-1024x373.png 1024w, /static/img/blog/2015/08/ws-plugin-6-300x109.png 300w, /static/img/blog/2015/08/ws-plugin-6.png 1643w" sizes="(max-width: 1024px) 100vw, 1024px" /></a>

Now, setup functions to render the input fields are used. These input fields are linked by their name tag, this name tag will match the same name defined within the &#8216;add_settings_field()&#8217; function.

<a href="/static/img/blog/2015/08/ws-plugin-7.png"><img loading="lazy"  src="/static/img/blog/2015/08/ws-plugin-7-1024x318.png" alt="ws-plugin-7" width="1024" height="318" srcset="/static/img/blog/2015/08/ws-plugin-7-1024x318.png 1024w, /static/img/blog/2015/08/ws-plugin-7-300x93.png 300w, /static/img/blog/2015/08/ws-plugin-7.png 2011w" sizes="(max-width: 1024px) 100vw, 1024px" /></a>

Lastly an &#8216;add_action()&#8217; function is triggered before any other hook when a user access the admin area, this will run the &#8216;display_options()&#8217; which will add and register the setting fields.

<a href="/static/img/blog/2015/08/ws-plugin-8.png"><img loading="lazy"  src="/static/img/blog/2015/08/ws-plugin-8.png" alt="ws-plugin-8" width="893" height="96" srcset="/static/img/blog/2015/08/ws-plugin-8.png 893w, /static/img/blog/2015/08/ws-plugin-8-300x32.png 300w" sizes="(max-width: 893px) 100vw, 893px" /></a>

Before we finish, the &#8216;style.css&#8217; document requires some styling rules. These CSS rules are only used to style the input and text area widths and heights.
